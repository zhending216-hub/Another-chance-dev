# 研究报告: src/app/layout.tsx

## 概述

该文件是 Next.js 应用的根布局组件，定义了全局 HTML 结构和元数据。

## 用途

作为整个应用的布局容器，设置页面语言为中文，导入全局样式，定义 SEO 元数据。

## 导出

- `metadata`：页面标题和描述。
- `RootLayout`：根布局组件。

## 依赖

- `next`（`Metadata` 类型）。
- `./globals.css`：全局样式文件。

## 核心逻辑

1. 设置 HTML 语言为 `zh-CN`。
2. 定义元数据：标题"古事 - 分叉故事续写平台"，描述"基于历史/经典故事的关键片段和人物产生分叉剧情的故事续写平台"。
3. 导入全局 CSS。
4. `body` 设置 `min-h-screen`。

结构极其简洁，是一个标准的 Next.js App Router 根布局。

## 数据流

```
RootLayout → <html lang="zh-CN"> → <body> → {children}
```

## ChronosMirror 升级改进点

### 1. 角色建模
- 根布局可添加全局的角色面板 Provider（Context），在整个应用中共享角色状态。
- 可添加全局的"角色选择器"浮窗。

### 2. 时间轴校验
- 可添加全局的时间轴 Context Provider。
- 布局中可嵌入全局时间轴导航栏。

### 3. MCP 维基百科
- 布局级别可初始化 MCP 连接配置。
- 可添加全局的"历史知识库"搜索功能。

### 4. 节奏控制
- 可添加全局的阅读设置（字体大小、行间距、主题色）。
- 可添加暗色模式支持。
- 可添加 PWA 配置，支持离线阅读。
- 可优化字体加载策略（中文字体较大，应按需加载）。
