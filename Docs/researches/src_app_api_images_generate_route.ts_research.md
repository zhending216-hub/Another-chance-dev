# src/app/api/images/generate/route.ts 研究报告

## 1. 文件用途

这是"古事"项目的 AI 图片生成 API 路由，负责接收前端请求并调用 AI 模型为故事段落生成配图。它是叙事可视化链条中的关键一环，将文字描述转化为视觉呈现。

## 2. 导出

- `POST`：处理图片生成请求
- `OPTIONS`：返回支持的图片参数选项（CORS 预检）

## 3. 依赖

- `next/server`：NextRequest、NextResponse
- 环境变量：AI_IMAGE_PROVIDER、AI_IMAGE_API_KEY、AI_IMAGE_MODEL、AI_IMAGE_BASE_URL

## 4. 核心逻辑

**POST 请求处理**：
1. 解析请求体：`segmentId`、`prompt`、`style`、`size`（默认 512x512）、`quality`（默认 standard）
2. 参数验证：segmentId 和 prompt 为必填
3. 读取 AI 图片生成配置（支持 OpenAI 兼容 API）
4. 当前实现为模拟返回（placeholder），未实际调用 AI API
5. 返回生成结果：imageId、imageUrl、metadata

**OPTIONS 请求处理**：
- 返回支持的方法、尺寸、质量、风格列表
- 尺寸：256x256 到 1792x1024
- 风格：realistic（写实）、artistic（艺术）、cartoon（卡通）、historical（历史）、fantasy（奇幻）

**关键设计特点**：
- Provider 抽象层：支持 OpenAI 兼容的多种 AI 图片服务
- 模拟实现：当前是占位符，返回固定的 placeholder 图片
- 风格参数：预留了 5 种图片风格，适配历史故事的不同视觉需求

## 5. 数据流

前端请求 → POST /api/images/generate → 参数验证 → 读取 AI 配置 → （当前：模拟生成）/（未来：调用 AI API） → 返回图片 URL 和元数据。OPTIONS 用于前端查询支持的能力参数。

## 6. ChronosMirror 升级改进点

### 角色建模
- **角色一致性生成**：当前 prompt 是自由文本，无法保证同一角色在不同段落中的视觉一致性。升级后应：
  - 引入角色视觉参考图（character reference），作为图片生成的额外参数
  - 使用 AI 的 image-to-image 或 ControlNet 技术保持角色外观一致
  - 在 prompt 中自动注入角色描述（外貌、服饰、特征）
- **角色表情/姿态**：根据角色在当前段落的状态（愤怒、悲伤、威严）自动调整 prompt 中的表情描述

### 时间轴校验
- **历史准确性图片**：图片生成应受时间轴校验约束。例如"荆轲刺秦王"的配图必须符合秦朝服饰和宫殿风格，不能出现明显的历史错误。
- **时期风格映射**：根据段落的历史时期自动选择图片风格参数：
  - 先秦 → historical（古朴风格）
  - 唐宋 → artistic（华丽风格）
  - 神话传说 → fantasy（奇幻风格）
- **校验反馈可视化**：时间轴校验发现的历史冲突可以通过生成"标注版"图片直观展示

### MCP 维基百科
- **维基百科图片参考**：从维基百科条目中提取历史图片作为 AI 生成的参考素材。例如荆轲刺秦王的维基页面可能包含相关的古画、文物照片。
- **历史场景还原**：利用维基百科中的历史场景描述（建筑风格、服饰特征、兵器形制）丰富 prompt，提升生成图片的历史准确性。
- **来源标注**：生成的图片元数据中应记录使用了哪些维基百科参考，便于用户追溯。

### 节奏控制
- **节奏驱动的图片风格**：叙事节奏应影响图片生成策略：
  - 高潮段落（紧张度高）→ 高对比度、动态构图
  - 铺陈段落（节奏慢）→ 柔和色调、静态构图
  - 过渡段落 → 渐变色调、流动构图
- **生成速度与节奏匹配**：快速推进的叙事段落使用更小的图片尺寸（256x256，快速生成），高潮段落使用高分辨率（1024x1024），实现"节奏感"的视觉体现。
- **批量预生成**：对于已确定的叙事路径，可以提前异步生成后续段落的配图，避免用户阅读时的等待。

### 总体改进建议
1. 实现真实的 AI API 调用（替换模拟代码）
2. 增加图片生成队列和重试机制
3. 增加生成结果缓存（相同 prompt 不重复生成）
4. 支持图片编辑（局部修改、风格迁移）
5. 增加内容安全过滤（历史暴力场景的适当处理）
6. 增加图片生成进度推送（SSE）
7. 支持用户自定义角色外观模板
