# 研究报告: src/app/create/page.tsx

## 概述

该文件是故事创建页面，提供"模板故事"和"自定义创作"两种模式。模板故事预设了三个经典历史事件（荆轲刺秦王、赤壁之战、玄武门之变），每个模板提供三个思考方向供用户选择。

## 用途

作为用户创建新故事的入口页面。通过精心设计的 UI 引导用户选择历史事件、思考方向，降低创作门槛。

## 导出

- `CreateStoryPage` 默认导出页面组件。

## 依赖

- React（`useState`）。
- `next/navigation`（`useRouter`）。
- `next/link`（`Link`）。
- 调用 `/api/stories` POST 接口创建故事。

## 核心逻辑

### 模板系统
- 三个预设模板：荆轲刺秦王（战国）、赤壁之战（三国）、玄武门之变（唐）。
- 每个模板包含：`id`、`title`、`description`、`era`、`icon`、`gradient`、`difficulty`、`time`、`prompts`。
- 模板仅是前端数据，未持久化——创建时仅传递 `title` 和 `description`。

### 故事类型
- 四种类型：历史分叉、神话传说、文学名著、原创故事。
- 目前类型选择仅影响 UI 展示，不影响 AI 生成逻辑。

### 创建流程
1. 用户选择故事类型。
2. 选择模板或自定义（Tab 切换）。
3. 模板模式：选择模板 → 选择思考方向 → 点击"开始这个故事"。
4. 自定义模式：填写标题、描述、作者、提示词 → 点击"创建故事"。
5. 创建成功后跳转到故事详情页。

### UI 设计
- 中国风视觉设计：纸张质感背景、金色边框、朝代标签。
- 模板卡片带渐变色头部和图标。
- 思考方向选择使用红色高亮主题。

## 数据流

```
CreateStoryPage → 选择模板/自定义 → fetch POST /api/stories
→ 成功 → router.push(/story/[id])
```

## ChronosMirror 升级改进点

### 1. 角色建模
- 模板故事应预置角色列表。创建时自动初始化角色数据。
- 创建页面可展示"出场人物"预览。
- 自定义创建时可手动添加角色。

### 2. 时间轴校验
- 模板应包含精确的时间信息（年月日）。
- 创建故事时应初始化时间轴。
- 自定义创建时可选择时间范围（朝代滑块）。

### 3. MCP 维基百科
- 模板选择后，可通过 MCP 查询维基百科展示历史事件的详细背景。
- 可在模板卡片中添加"了解更多"按钮，链接到维基百科页面。
- 创建时可展示"历史时间线"，帮助用户了解事件背景。

### 4. 节奏控制
- 模板应包含"建议故事长度"和"节奏预设"（如"紧凑型"、"史诗型"）。
- 创建时可选择"叙事风格"（文言文、半文半白、白话文）。
- 应支持更多模板（目前仅 3 个），可按朝代分类。
- 难度和时间标注目前仅是 UI 展示，应影响实际的 AI 生成参数。
