# 研究报告: src/app/story/[id]/page.tsx

## 概述

该文件是古事平台的核心页面——故事阅读页。它集成了段落展示、流式续写、分支创建、分支切换等全部核心功能，是用户与故事互动的主要界面。

## 用途

提供完整的故事阅读和互动体验：浏览段落、续写故事、创建分支、切换分支查看不同剧情线。

## 导出

- `StoryDetailPage` 默认导出页面组件。
- 内部组件：`BranchDialog`（分叉对话框）。

## 依赖

- React（`useState`、`useEffect`、`useCallback`）。
- `next/link`（`Link`）。
- 调用 `/api/stories/[id]` GET、`/api/stories/[id]/tree` GET、`/api/stories/[id]/segments` GET、`/api/stories/[id]/stream-continue` POST、`/api/stories/[id]/branch` POST。

## 核心逻辑

### 数据加载
1. 并行加载故事详情和树结构。
2. 加载当前分支的有序段落。
3. 分支切换时重新加载段落。

### 续写功能
1. 点击"续写故事"按钮。
2. 调用 `stream-continue` POST 接口。
3. 通过 `ReadableStream` 逐 chunk 读取 SSE 数据。
4. 实时更新 `newContent` 状态，渲染流式内容（带闪烁光标）。
5. 流结束后重新加载段落和树结构。

### 分支功能
1. 每个段落都有"分叉"按钮。
2. 点击后弹出 `BranchDialog` 对话框。
3. 用户选择预设方向或输入自定义方向。
4. 调用 `branch` POST 接口创建分支。
5. 分支创建过程有三步进度展示（构思→生成→保存）。
6. 完成后刷新段落和树结构。

### UI 设计
- 顶部导航栏：返回链接、当前路径、故事标题。
- 分支切换栏：主线 + 所有分支的 Tab 按钮。
- 故事标题区：装饰性分隔符 + 大标题 + 描述。
- 时间轴样式：左侧金色装饰线 + 圆形节点（分叉点为红色脉冲动画）。
- 段落卡片：标题、内容、分叉点标记、分叉按钮。
- 流式内容区：蓝色主题，带闪烁光标。
- 底部操作栏：续写按钮。

### 分支对话框
- 四个预设方向（战争策略、外交途径、内政发展、寻求盟友）。
- 自定义方向输入框。
- 创建过程的三步进度展示。

## 数据流

```
StoryDetailPage
→ 加载 story + tree + segments
→ 渲染时间轴 + 段落卡片
→ 续写: stream-continue SSE → 实时渲染 → 保存
→ 分支: BranchDialog → branch POST → 刷新
→ 切换分支: switchBranch → reload segments
```

## ChronosMirror 升级改进点

### 1. 角色建模
- 段落卡片应显示该段落中出现的角色列表。
- 可添加侧边栏"角色面板"，实时显示当前分支中所有角色的状态。
- 分支创建时应展示角色在该分支中的预期变化。
- 可添加角色对话视图——按角色筛选段落。

### 2. 时间轴校验
- 左侧装饰线应变为真正的时间轴，标注每个段落的叙事时间。
- 时间轴可视化应显示不同分支的时间进度差异。
- 续写前应校验时间轴合理性（如不应超过历史事件的时间范围）。
- 可添加"时间跳转"功能，快速定位到特定时间点的段落。

### 3. MCP 维基百科
- 段落中提及的历史名词（人名、地名、事件名）应可点击查看维基百科摘要。
- 可添加"历史对照"面板——并排展示故事内容和真实历史记录。
- 分支创建时，AI prompt 应注入 MCP 查询的历史背景知识。

### 4. 节奏控制
- 段落过长时应支持折叠/展开。
- 应添加"阅读进度"指示器（当前阅读位置 / 总段落数）。
- 流式续写应支持"暂停"和"跳过"功能。
- 分支对话框的预设方向应根据当前故事内容动态生成（而非固定的四个选项）。
- 应添加"自动续写"模式——AI 连续生成多段内容，形成完整的叙事弧。
- 页面应优化移动端体验（当前设计偏向桌面端）。
