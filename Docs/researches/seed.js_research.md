# seed.js 研究报告

## 1. 文件用途

seed.js 是"古事"项目的数据库种子文件，用于初始化示例故事数据。它填充了三个经典历史事件（荆轲刺秦王、赤壁之战、玄武门之变）的故事和段落数据，为开发和演示提供基础内容。

## 2. 导出

不导出任何内容。直接执行 `node seed.js` 运行种子脚本。

## 3. 依赖

- `./src/lib/simple-db`：自定义的 JSON 文件数据库模块
  - `storiesStore`：故事存储
  - `segmentsStore`：段落存储
  - `branchesStore`：分支存储（已导入但未使用）

## 4. 核心逻辑

**数据结构**：
- 故事（Story）：id、title、description、author、createdAt、updatedAt
- 段落（Segment）：id、title、content、order、isBranchPoint、storyId、imageUrls

**三个示例故事**：
1. **荆轲刺秦王**（公元前227年）：2个段落 - 序幕、面见秦王（分支点）
2. **赤壁之战**（东汉末年）：2个段落 - 曹操南下、联军 formed（分支点）
3. **玄武门之变**（公元626年）：2个段落 - 兄弟矛盾、玄武门兵变（分支点）

**执行流程**：
1. 导入 simple-db 存储实例
2. 定义三个故事的段落模板
3. `createStory()` 为每个故事创建记录（使用 `Date.now()` 作为 ID）
4. `createSegment()` 为每个故事创建关联段落
5. 控制台输出创建进度

**关键设计**：
- 段落的 `isBranchPoint` 标记了故事分叉点，用户可在此处选择不同的故事走向
- `imageUrls` 预留了图片字段，但种子数据中均为空数组
- `branchesStore` 已导入但未使用，暗示分支功能尚未完全实现

## 5. 数据流

seed.js → simple-db → JSON 文件读写 → data/ 目录下的 JSON 文件。每个故事获得唯一 ID，段落的 `storyId` 关联到对应故事。

## 6. ChronosMirror 升级改进点

### 角色建模
- **角色数据种子**：当前种子数据完全没有角色信息。升级后需要增加角色种子数据，包括：
  - 荆轲（刺客，勇敢果断）、秦王嬴政（帝王，威严多疑）
  - 曹操（枭雄，雄才大略）、诸葛亮（智者，运筹帷幄）
  - 李世民（秦王，英明果断）、李建成（太子，猜忌多疑）
- **角色关系数据**：需要种子化角色之间的关系（如荆轲-太子丹的君臣关系、曹操-孙权的敌对关系），为角色建模提供初始数据。
- **角色状态演变**：每个段落应包含参与角色的状态变化，如荆轲从"准备中"变为"行刺中"。

### 时间轴校验
- **精确时间标记**：当前段落只有 `order` 排序，没有时间信息。应增加 `timelineDate`（如"公元前227年"）、`timelineEra`（如"战国末期"）等字段。
- **历史时间线种子**：应预置完整的历史时间线数据，包括每个朝代的关键事件节点，作为时间轴校验的参照基准。
- **分支时间线**：每个分支点后的不同走向应标注"真实历史"vs"虚构分支"，便于时间轴校验区分。

### MCP 维基百科
- **知识库关联**：每个故事应关联对应的维基百科条目 ID，如荆轲刺秦王关联 `https://zh.wikipedia.org/wiki/荆轲刺秦王`。种子数据中应预填这些关联。
- **历史背景数据**：应增加更丰富的历史背景段落，并标注来源（维基百科引用），为 MCP 集成提供内容基础。

### 节奏控制
- **段落节奏标记**：当前段落没有节奏属性。应增加 `pace`（节奏速度：fast/normal/slow）、`tension`（紧张度：1-10）等字段。
- **叙事节奏模板**：种子数据应包含不同节奏模式的示例：
  - 荆轲刺秦王：慢速铺陈 → 快速高潮（行刺瞬间）
  - 赤壁之战：中速推进 → 多线并行 → 高潮爆发（火烧赤壁）
- **段落数量扩充**：每个故事仅 2 个段落过于简略，建议扩充至 8-12 个段落以充分展示节奏变化。

### 总体改进建议
1. 重构为 TypeScript 模块，配合 Prisma schema
2. 增加角色、关系、时间线等维度的种子数据
3. 支持从外部 JSON/YAML 文件导入种子数据（而非硬编码）
4. 增加 idempotent 逻辑（重复运行不会创建重复数据）
5. 增加种子数据的验证步骤（确保关联关系正确）
6. 为每个故事增加完整的 8-12 段叙事线，充分展示所有功能
